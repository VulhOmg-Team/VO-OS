# Contributing to VO-OS

Thank you for your interest in contributing to VO-OS.

VO-OS is an open source project built together with the Linux community.

Everyone can contribute through code, documentation, testing, translations, design, and ideas.

---

# Ways to Contribute

## Code

Developers can help with:

- VO Core;
- VO Center;
- system tools;
- modules;
- installer;
- testing tools.

## Documentation

Help improve:

- technical documentation;
- user guides;
- translations;
- tutorials.

## Testing

Test:

- installations;
- updates;
- hardware compatibility;
- performance;
- stability.

## Design

Contribute:

- themes;
- icons;
- user interface ideas;
- visual improvements.

---

# Before Contributing

Before starting work:

1. Check existing issues.
2. Read project documentation.
3. Discuss large changes before implementation.
4. Make sure your contribution follows project principles.

---

# Development Process

The recommended workflow:

1. Fork the repository.
2. Create a new branch.
3. Make your changes.
4. Test your changes.
5. Create a pull request.

Example branch names:

```
feature/vo-center-module-manager

bugfix/installer-network-error

docs/update-architecture
```

---

# Pull Requests

A good pull request should include:

- clear description;
- reason for the change;
- testing information;
- related issues if available.

Large changes may require discussion before merging.

---

# Code Requirements

Code should be:

- readable;
- documented;
- secure;
- maintainable.

Avoid:

- unnecessary complexity;
- copied code without license information;
- breaking existing functionality.

---

# Open Source Respect

VO-OS is built on the work of many Linux contributors.

Contributors must:

- respect software licenses;
- keep author information where required;
- document external technologies;
- follow open source requirements.

---

# Community

VO-OS welcomes contributors of all experience levels.

Beginners can help with:

- testing;
- documentation;
- translations;
- feedback.

Experienced contributors can help with:

- architecture;
- programming;
- security;
- optimization.

Together we continue the development of Linux.
